## 执行摘要

Mycel Mesh 是一款基于 WireGuard 协议的企业级虚拟组网解决方案，将分布在不同地理位置的设备和服务器无缝连接到虚拟局域网中。通过自研的 NAT 穿透技术和智能路由算法，实现高达 85% 以上的直连成功率，为企业和个人用户提供安全、高效、易用的虚拟网络服务。

### 核心价值

| 价值维度 | 用户收益 |
|---------|---------|
| **降低成本** | 零依赖部署，单二进制文件，无需复杂配置 |
| **提升效率** | 节点自动加入，IP 自动分配，秒级连接建立 |
| **保障安全** | WireGuard ChaCha20-Poly1305 加密，端到端安全 |
| **可视管理** | Web UI 实时监控，32+ 指标，7 条告警规则 |

## 一、产品概述

### 1.1 产品定位

Mycel Mesh 定位于**轻量级虚拟组网工具**，解决以下核心痛点：

1. **跨网络访问难** - 设备分布在不同网络环境，无法直接通信
2. ** NAT 穿透复杂** - 传统方案穿透率低，依赖中继服务器
3. **管理成本高** - 需要手动配置 IP、路由、防火墙规则
4. **监控能力弱** - 缺乏可视化工具，无法实时掌握网络状态

### 1.2 目标用户

| 用户类型 | 典型场景 | 核心需求 |
|---------|---------|---------|
| **中小企业** | 多地办公网络互联 | 低成本、易管理、安全可靠 |
| **开发者** | 远程开发环境访问 | 快速部署、端口映射、调试方便 |
| **运维工程师** | 服务器集群管理 | 批量管理、监控告警、访问控制 |
| **技术爱好者** | 家庭实验室组网 | 简单易用、文档完善、社区活跃 |

### 1.3 技术优势

| 技术指标 | Mycel Mesh | 传统方案 |
|---------|-----------|---------|
| NAT 穿透率 | >85% | 60-70% |
| 连接建立时间 | <100ms | 1-3s |
| 加密算法 | ChaCha20-Poly1305 | AES-256-GCM |
| 资源占用 | <50MB 内存 | 100-200MB |
| 部署复杂度 | 单文件 | 多组件 |

## 二、核心功能

### 2.1 节点管理

**功能描述**: 自动发现和管理网络中的节点

- **自动注册** - 节点启动后自动向 Coordinator 注册
- **IP 自动分配** - DHCP 式 IP 分配，无需手动配置
- **状态监控** - 实时显示节点在线/离线状态
- **延迟检测** - 自动测量节点间延迟

**用户价值**: 零配置组网，降低运维成本

### 2.2 NAT 穿透

**功能描述**: 自研 STUN + UDP Hole Punching 技术

- **NAT 类型检测** - 自动识别 Full Cone、Symmetric 等类型
- **智能打洞** - 根据 NAT 类型选择最优穿透策略
- **中继兜底** - 穿透失败自动切换到中继模式

**用户价值**: 85%+ 直连率，降低中继服务器成本

### 2.3 访问控制 (ACL)

**功能描述**: 精细化的网络访问控制

- **规则管理** - 支持允许/拒绝规则
- **端口粒度** - 可精确到具体端口和协议
- **优先级** - 规则按优先级顺序匹配

**用户价值**: 满足企业安全合规要求

### 2.4 多子网支持

**功能描述**: 支持创建和管理多个虚拟子网

- **子网隔离** - 不同子网间默认隔离
- **子网路由** - 可配置子网间路由规则
- **CIDR 灵活配置** - 支持 /8 到 /32 任意掩码

**用户价值**: 满足复杂网络架构需求

### 2.5 监控告警

**功能描述**: 基于 Prometheus + Grafana 的监控体系

- **32+ 指标** - 节点状态、流量、延迟、NAT 类型等
- **3 个 Dashboard** - 节点总览、流量监控、NAT 穿透
- **7 条告警规则** - 节点离线、高延迟、穿透失败等

**用户价值**: 实时掌握网络健康状态

### 2.6 Web UI

**功能描述**: 基于 React + TypeScript 的可视化管理界面

- **节点列表** - 查看所有节点状态和详情
- **流量图表** - 实时显示流量趋势
- **配置管理** - Web 界面配置 ACL、子网等

**用户价值**: 降低使用门槛，提升管理效率

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    Mycel Mesh 架构                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐         ┌─────────────┐               │
│  │  Web UI     │         │  mycelctl   │               │
│  │  (React)    │         │   (CLI)     │               │
│  └──────┬──────┘         └──────┬──────┘               │
│         │                       │                       │
│         └───────────┬───────────┘                       │
│                     │                                   │
│            ┌────────▼────────┐                          │
│            │  Coordinator    │                          │
│            │  (控制面)        │                          │
│            │  - gRPC/HTTP    │                          │
│            │  - 节点管理      │                          │
│            │  - IP 分配       │                          │
│            │  - ACL 服务      │                          │
│            │  - Metrics      │                          │
│            └────────┬────────┘                          │
│                     │                                   │
│         ┌───────────┼───────────┐                       │
│         │           │           │                       │
│    ┌────▼────┐ ┌────▼────┐ ┌────▼────┐                 │
│    │ Agent 1 │ │ Agent 2 │ │ Agent N │                 │
│    │ (节点)  │ │ (节点)  │ │ (节点)  │                 │
│    └────┬────┘ └────┬────┘ └────┬────┘                 │
│         │           │           │                       │
│         └───────────┴───────────┘                       │
│                     │                                   │
│            ┌────────▼────────┐                          │
│            │   WireGuard     │                          │
│            │   (数据面)      │                          │
│            └─────────────────┘                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 组件说明

| 组件 | 职责 | 技术栈 |
|------|------|--------|
| **Coordinator** | 控制面核心，负责节点管理、IP 分配、ACL、路由 | Go + gRPC + PostgreSQL |
| **Agent** | 数据面节点，负责建立 WireGuard 隧道 | Go + WireGuard-go |
| **mycelctl** | CLI 命令行工具，用户交互入口 | Go + Cobra |
| **Web UI** | 可视化管理界面，实时监控 | React + TypeScript + Ant Design |

### 3.3 数据流

```
用户请求 → mycelctl/Web UI → Coordinator (gRPC) → Agent (WireGuard) → 目标节点
                                                            ↓
                                                    直连 (85%+) 或 中继
```

## 四、部署方案

### 4.1 单机部署 (开发/测试)

**适用场景**: 个人开发、小范围测试

**资源需求**:
- CPU: 1 核
- 内存：512MB
- 磁盘：1GB

**部署步骤**:
```bash
# 1. 下载
git clone https://github.com/mycel/mesh.git
cd mesh

# 2. 构建
make build

# 3. 启动 Coordinator
./bin/coordinator --init

# 4. 启动 Agent
./bin/agent --join <token>
```

### 4.2 集群部署 (生产环境)

**适用场景**: 企业生产环境

**资源需求**:

            ┌────────────────┼────────────────┐
            │                │                │

## 五、使用场景

### 5.1 场景一：多地办公网络互联

**背景**: 公司在北京、上海、深圳有三个办公室，需要实现内部网络互通。

**方案**:
1. 在北京部署 Coordinator
2. 每个办公室部署一个 Agent
3. 配置 ACL 规则限制访问权限

**效果**:
- 三地办公室网络互通
- 共享内部资源 (文件服务器、打印机等)
- 访问控制保障安全

### 5.2 场景二：远程开发环境

**背景**: 开发者需要访问公司内部的开发环境。

**方案**:
1. 开发者本地安装 Agent
2. 使用 token 加入公司网络
3. 通过虚拟 IP 访问开发环境

**效果**:
- 无需 VPN，配置简单
- 自动 NAT 穿透，无需公网 IP
- 可限制访问范围

### 5.3 场景三：混合云组网

**背景**: 企业同时使用阿里云、腾讯云和自建 IDC，需要统一网络。

**方案**:
1. 每个云/IDC 部署一个 Agent
2. 创建多个子网 (生产、测试、开发)
3. 配置子网间路由规则

**效果**:
- 混合云统一管理
- 子网隔离保障安全
- 可视化监控流量

### 5.4 场景四：IoT 设备管理

**背景**: 管理分布在全国的 IoT 设备。

**方案**:
1. 每个 IoT 设备安装 Agent
2. 使用隔离子网
3. 通过 Coordinator 统一管理

**效果**:
- 设备自动上线
- 远程管理维护
- 流量统计和告警

## 七、安全特性

### 7.1 加密算法

| 算法 | 用途 | 强度 |
|------|------|------|
| ChaCha20-Poly1305 | 数据加密 | 256 位 |
| Curve25519 | 密钥交换 | 256 位 |
| BLAKE2s | 哈希 | 256 位 |

### 7.2 认证机制

- **Token 认证** - 节点加入需要有效 token
- **JWT 认证** - API 访问需要 JWT 令牌
- **失败锁定** - 登录失败 5 次自动锁定

### 7.3 访问控制

- **ACL 规则** - 基于源/目标、端口、协议的访问控制
- **子网隔离** - 不同子网默认隔离
- **审计日志** - 记录所有管理操作
